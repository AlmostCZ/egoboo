Proposal

This are the descriptions for the different types of characters in EGOBOO
The data structs are given in XML
The effects are at the heart of the matter:
- Items have effects
- Monster-attacks have effects
- Spells have effects

Effects can be temporary or permanent

Tag: <effect>

EFFECT
XML-Tag value Description
<use> char EFFTYPE_ATTACK: Attacks a game character
EFFTYPE_HEAL: Heals a game character
EFFTYPE_SKILL: Changes skill of a game character
EFFTYPE_DEFENSE: Changes defense/vulnerability of a game character for an effect
EFFTYPE_ABILITY Changes the value of an ability (ABILITY_MANA etc.)
EFFTYPE_MISC: E.g. bump money
<type> char e. g. EFFECT_ZAP
Attack is modified by the defense / vulnerability of a character for this type
If the character has 100% defense, then the attack has no effect

If EFFECT_NONE, then there is no defense against this effect
<dice> char / char / char Value of attack / heal:
[Number of throws (0..10)]
[Number of sides (1..20)]
[Add to thrown dice value (0..100)] Can be negative
If the number of throws is zero, then the added value is used as fixed value
<duration> float Time in seconds for this effect (float)
> 0: Timer ist set in character, effect lasts for this time. After that, it is removed from the character
0: Effect is once and permanent, no timer is set
<repeat> char How many time the timer is repeated:
> 0: Timer is restarted and effect is applied each time the timer is restarted (e. g. for poison effects)
0: Timer is runned once
< 0: Timer is repeated until removed by another effect / spell whatever
