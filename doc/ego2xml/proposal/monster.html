Proposal

Definition of a monster for spawning

MONSTER
XML-Tag Description
<cooldown> Time between attacks
<type> Which MONSTER_TYPE (Name)
<model> Name of model to use
<initstate> State at spawn time
<attacks> Number of attacks or spells
<pocketitem> Name of item in pocket, is dropped if monster is killed
<weapon> Weapon in hand, is dropped if monster is killed
<skin_no> Which skin to use, if the model has multiple skins [1 .. num_skins]
0: Use random skin, if more than one is available

MONSTER_TYPE
XML-Tag Description
<name> Name of type, given a spawning monster, defines it abilities and how it behaves in the game
<armorclass> Base defense, attack points are reduced by armor class
<dice> Life at creation time DICE(numthrow, sides, addvalue)
<attackcnt> Number of attacks
<attack_list>
<attack>
<type> ATT_POSION, ATT_TURNSTONE, ... (like items)
<dice> Dice to roll for each attack (numthrow, numsides, addvalue)
<attack>
</attack_list>
<size> Size of monster
 0: A quarter of a tile
 1: Full tile
 2: Half of a tile
<attacksound> Number of sound to play if monster attacks
<walksound> Sound to play as the monster walks
<killexperience> Experience the player gets if he kills that monster
<thcount> How many weapons the monster has to throw (Item 'in Hand')
After the items are used up, the monster needs to change to melee battle
